# 05 集成与验收日志

集成线程使用这个文件记录合并、冲突、联调、测试、发布前检查。

| 日期 | 集成范围 | 来源线程/分支 | 处理内容 | 验证方式 | 结果 | 遗留问题 |
| --- | --- | --- | --- | --- | --- | --- |
| YYYY-MM-DD | 示例：登录模块 | thread-auth | 合并登录接口和前端登录态 | npm test | 通过 | 无 |

## 集成线程职责

- 读取 `03-task-board.md` 和 `04-thread-handoff.md`
- 合并各业务线程变更
- 处理冲突
- 执行全量测试
- 检查主链路是否贯通
- 更新 `01-feature-map.md`、`02-milestones.md`、`03-task-board.md`
- 记录真实未完成项，不把外部依赖写成已完成

## 发布前检查

- 功能点是否全部有状态
- P0/P1 是否全部 `DONE`
- 阻塞项是否有明确原因
- 外部服务是否真实接入
- 数据库迁移是否完整
- OpenAPI 或接口文档是否同步
- README 或部署文档是否同步
- 测试命令和结果是否记录

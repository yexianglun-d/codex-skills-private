## 拆分规则

- 一个功能点必须能被验收。
- 不把多个大功能塞进同一个 ID。
- 每个功能点必须有明确模块、优先级、依赖和验收标准。
- 技术债、文档同步、测试补齐也可以作为功能点或工程任务进入矩阵。

## 优先级规则

| 优先级 | 含义 |
| --- | --- |
| P0 | 没有它主链路不可用 |
| P1 | 影响核心体验或完整交付 |
| P2 | 增强能力、管理能力、边界完善 |
| P3 | 优化、体验提升、低风险补充 |

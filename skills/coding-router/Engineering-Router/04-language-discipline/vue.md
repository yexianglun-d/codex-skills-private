# Vue Route

Use for Vue 2, Vue 3, Nuxt, Vite Vue, uni-app Vue, single-file components, Composition API, Options API, Pinia, Vuex, router, forms, API calls, component state, style, build, lint, or type-check work.

## Route

- Use `vue-code-discipline`.
- Add `root-cause-fixer` for bugs, failing tests, regressions, or incorrect runtime behavior.
- Use `$ai-design-router` first when the task is primarily visual design, UX, layout quality, Figma, or visual QA.
- Use Context7 for current library, framework, component-library, API, or CLI details.

## Focus

- Follow existing project component structure, API client, store, router, style system, and directory conventions.
- Keep components responsible and predictable.
- Verify with lint, type-check, build, tests, or browser checks that match the change.

## Do Not

- Mix Options API and Composition API against project convention.
- Add visual-design preferences when the task is a data-flow or logic fix.
- Introduce a new state library, UI library, or CSS approach for a local issue.

# Vue 评审与重构清单

用于 Vue 代码评审、组件重构、页面治理和质量检查。

## 组件结构

- 页面组件是否只负责路由级编排，复杂业务逻辑是否已按项目习惯拆分？
- 基础组件、业务组件、页面组件边界是否清楚？
- `props`、`emits`、slots、`v-model` 语义是否清楚，是否避免隐式耦合？
- 组件命名、文件命名、目录分层是否符合当前项目。

## 状态与数据流

- 单向数据流是否清晰，是否存在直接改 `props`、跨层级强耦合或隐藏副作用？
- store/composable/API service 的职责是否清楚，是否把请求、缓存、转换、展示逻辑混在组件里？
- computed 是否用于派生状态，watch 是否只处理必要副作用？
- 是否存在重复请求、竞态覆盖、卸载后 set state、重复提交或快速切换状态错乱。

## 模板与交互

- `v-for` 是否有稳定 `key`。
- 是否避免同节点 `v-if` + `v-for`。
- loading、empty、error、disabled、权限不可见、移动端适配等状态是否齐全。
- 表单校验、提示文案、提交防抖或禁用逻辑是否符合项目既有模式。
- 键盘可达性、语义标签、基础可访问性是否没有明显倒退。

## 样式与视觉一致性

- 是否复用项目已有变量、设计 token、组件库、布局工具和响应式断点。
- 是否避免卡片套卡片、随意阴影、随意圆角、临时 inline style、全局样式泄漏。
- scoped / CSS Modules / Tailwind / UnoCSS / Less / Sass 是否遵循项目既有方案。
- 是否没有为了局部页面引入新的审美系统或 UI 库。

## 验证

- 是否运行与改动范围匹配的 `lint`、`type-check`、测试、构建或浏览器验证。
- 如果是 UI 行为改动，是否验证关键交互路径，而不只看静态页面能打开。
- 如果是小程序、SSR、移动端或多端项目，是否区分 H5、目标端构建产物和真实运行环境。

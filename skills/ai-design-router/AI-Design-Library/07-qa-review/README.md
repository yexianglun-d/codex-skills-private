# QA Review

Use for visual verification, screenshot review, interaction checks, responsive layout checks, and UI regression debugging.

## Installed Or Plugin Skills

- `playwright`
- `playwright-interactive`
- `build-web-apps:frontend-testing-debugging`
- `product-design:audit`
- `product-design:design-qa`

## Default Choice

- Use Playwright for local browser verification, screenshots, flows, and layout checks.
- Use frontend testing/debugging skill when a Build Web Apps workflow or local dev server is involved.
- Use product-design audit for broader UX flow critique.

## Do Not

- Claim visual quality from build success alone.
- Skip mobile or narrow viewport checks when layout is part of the task.
- Treat a single screenshot as enough for dynamic, animated, or interactive UI.

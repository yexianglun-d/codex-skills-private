# Product UX

Use for product direction, flows, MVP scope, UX review, onboarding, funnels, and screen structure before implementation.

## Installed Or Plugin Skills

- `product-co-creator`
- `product-design:get-context`
- `product-design:audit`
- `product-design:ideate`
- `product-design:image-to-code`

## Default Choice

- Use `product-co-creator` when the idea is not yet a build-ready UI task.
- Use product-design plugin skills when the task asks for UX audit, design alternatives, screenshots, or image-to-code workflows.

## Do Not

- Jump to frontend implementation before product flow is clear.
- Expand a small UI tweak into a full product roadmap.

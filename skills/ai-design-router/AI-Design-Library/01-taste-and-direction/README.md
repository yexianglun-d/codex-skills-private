# Taste And Direction

Use for visual direction, anti-template critique, design polish, and avoiding generic AI-looking UI.

## Installed Skills

- `taste-skill`
- `frontend-design`
- `ui-ux-pro-max`

## Default Choice

- Use `frontend-design` for normal high-quality UI creation.
- Add `taste-skill` when the user asks for stronger visual taste, premium feel, anti-template output, or critique.
- Keep `ui-ux-pro-max` explicit-only unless the user specifically asks for it or the task needs its private UI/UX knowledge base.

## Do Not

- Apply taste rules blindly to operational SaaS/admin tools.
- Redesign an existing product from scratch unless the user asks for overhaul.
- Mix multiple taste libraries by default.

# Design References

This folder vendors selected `DESIGN.md` references from `VoltAgent/awesome-design-md`.

Use it when the user asks for a named visual language such as Apple, Linear, Airbnb, Spotify, Stripe, Vercel, Notion, Figma, Raycast, Shopify, Supabase, or Cursor.

## How To Use

1. Read `catalog.md` to confirm the available reference.
2. Read a short route card under `design/` if it exists.
3. Read only the selected full source file under `awesome-design-md/design-md/<name>/DESIGN.md`.
4. Pair the reference with the relevant implementation skill:
   - General web UI: `frontend-design`
   - Vue project: `vue-code-discipline` + optional `frontend-design`
   - React/Next: React best-practices + optional `frontend-design`
   - Figma workflow: Figma skill + selected reference

## Guardrails

- Do not blend multiple brand references unless the user asks for a hybrid.
- Do not copy a brand identity verbatim for a real product. Use references as visual direction, not trademark replication.
- Existing project tokens, components, and brand rules override these references.
- If the chosen reference conflicts with the product category, explain the tradeoff briefly.
